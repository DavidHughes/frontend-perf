Auditing Performance

Browser DevTools

We are most familiar with Chrome and so all of the browser examples will follow suit. However, most of the tools demoed are available in all browsers.

Every browser has a powerful set of development tools (DevTools) lurking inside, waiting for you to unlock its full potential. Every type of browser performance problem, from page loads to animation jank, can be debugged directly in the browser. We will cover the most oft-used tools.

Activities

How many of these goals can you complete?

Browser DevTools

  1. Use the Network tab to see the total page weight and time until DOMContentLoaded event.
  2. Use the Network tab to find the heaviest resource on a web page.
  3. Use the Network tab to find the resource with the most latency.
  4. Look at the life cycle of the latent request and determine where the latency ocurred.

Auditing Services

  1. Use Google PageSpeed to analyze a web page. Use the information to make a plan for fixing the issues.
  2. Use WebPageTest to determine how long it takes a web page to start rendering using an iPhone 4 on 3G mobile connection.
  3. Use WebPageTest to determine the differences between a "cold" load (first-time visit) versus a repeat visit with browser cache primed.
  4. Use WebPageTest to generate a film-strip of a web page render.
  5. Use WhatDoesMySiteCost to roughly determine the cost of a web page in various countries.
